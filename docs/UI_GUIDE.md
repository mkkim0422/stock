# UI_GUIDE — Streamlit UI/UX 가이드 (정성을 다해)

## 비전공자 사용자 페르소나
- 한국어 1차. 영어 용어는 ()로 병기.
- 차트는 "보면 바로 이해" — 색·라인·툴팁.
- 숫자는 단위/통화 명시 (₩, $, %, 주, 거래일).
- 에러는 친절: "왜/무엇/다음에 무엇" 3단계로.

## UI/UX 원칙 (모든 페이지 공통)
1. **정보 위계** — 1번 큰 KPI 카드(총 평가액, 일간 손익) → 2번 표/차트 → 3번 액션 → 4번 로그.
2. **한 화면 한 목적** — 가상투자 화면은 "주문"에만 집중. 결과 분석은 별 페이지.
3. **색상 의미 일관**:
   - 🟢 초록 `#16a34a` — 수익/매수
   - 🔴 빨강 `#dc2626` — 손실/매도
   - 🔵 파랑 `#1f77b4` — 중립/액션
   - ⚫ 회색 `#6b7280` — 비활성/보조
4. **빈 상태 (Empty state)** — "아직 거래가 없어요. '매수' 탭에서 첫 거래를 해보세요." 같은 안내.
5. **로딩 인디케이터** — `st.spinner("불러오는 중...")` 모든 비동기.
6. **확인 단계** — 매수 클릭 → 요약 카드(수량/예상가/수수료) → 한 번 더 확인.
7. **숫자 포맷** — 천단위 콤마 (`f"{x:,.0f}"`), 통화 prefix, 소수점 2자리.
8. **위치 고정** — 사이드바에 항상: 모드, 평가액(KRW+USD), 마지막 갱신 시각.
9. **푸터 면책** — 모든 페이지 하단 작은 글씨로 면책 1줄.
10. **반응형은 무시** — Streamlit 기본 컬럼만 사용 (사용자 1명).

## 사이드바 표준 구성
```
[swing-advisor]
모드: 페이퍼
KRW: ₩10,000,000
USD: $10,000
환산총액: ₩23,500,000
마지막 갱신: 2026-05-27 23:55 KST
---
[페이지 네비]
```

## 페이지별 핵심 정보

### 1_Home.py 🏠 홈
- 환영 인사 + 면책 (큰 카드)
- 오늘의 수치: 평가액, 일간 손익, 보유 종목 수
- "지금 할 일" 3가지 버튼 (매수/매도/결과 보기)
- 시스템 상태 (DB OK, 데이터 시점, 모드)

### 2_Market.py 📈 시장 (placeholder)
- "Phase 2에서 활성됩니다" 안내 카드
- KOSPI/KOSDAQ/S&P500/NASDAQ 미리보기 (mock 값)

### 3_Stocks.py 🔍 종목 (placeholder)
- 종목 검색 박스 (작동 — symbols/master 기반)
- 검색 결과 표
- 상세는 "Phase 3에서 활성"

### 4_PaperTrading.py 💼 가상투자 (작동)
- **좌측**: 종목 검색 + 현재가(mock) + 주문폼 (수량, 매수/매도)
- **우측**: 주문 미리보기 카드 (수량, 예상가, 수수료, 거래세, 합계)
- 확인 버튼 → 체결 → 토스트 알림
- 하단: 최근 10건 거래 표

### 5_Results.py 📊 투자결과 (작동)
- 상단 KPI 4카드: 총 평가액 / 일간 손익 / 누적 수익률 / 보유 종목 수
- 자산 곡선 (plotly line chart, hover 툴팁)
- 보유 포지션 표: 종목, 수량, 평단, 현재가, 평가손익, 수익률(%)
- 손익 색상 자동 (초록/빨강)
- 다운로드: CSV 버튼

### 6_Backtest.py 🧪 백테스트 (placeholder)
- "Phase 4에서 활성" 안내

### 7_Settings.py ⚙️ 설정 (placeholder)
- 모드 표시 (readonly: paper)
- 환경변수 상태 (마스킹된 .env 키)
- DB 경로/사이즈

### 8_Help.py ❓ 도움말 (Phase 1)
- FAQ
- 면책 전문
- 외부 링크 (docs/ 안내)

## 파일명 규약 (재확인)
- 영문 + 번호 prefix만. 한글 파일명 절대 금지 (Streamlit URL 인코딩 문제).
- 페이지 안에서 `st.title("💼 가상투자")` 등 한글 표시.
